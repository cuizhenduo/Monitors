项目总结

围绕MTV的思想

项目存在的问题：
    1.使用同步的思想开发的
    2.单线程模式，造成阻塞
    3.同时请求8个url地址
    4.每个url地址平均响应的时间是2s
    5.等待上一个url响应成功以后，才会接着响应下一个url
    6.执行方式：串行执行
    7.总共用时：8 * 2 = 16s
    url -> url -> url -> url -> url -> url -> url -> url ->

解决阻塞的问题：
    1.使用异步的思想优化
    2.线程池模式，异步协程
    3.同时请求8个url地址
    4.每个url地址平均响应的时间是2s
    5.不需要等待上一个url响应成功以后，才会接着响应下一个url
    6.为每个请求分配一个线程
    7.执行方式：并行执行
    8.总共用时：2s
    url ->
    url ->
    url ->
    url ->
    url ->
    url ->
    url ->
    url ->
